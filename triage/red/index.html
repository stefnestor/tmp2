If querying live against the cluster, admins can filter-in to status with Elasticsearch API's filter_path (ref)

GET _cluster/health?filter_path=status

From an Elasticsearch diagnostic, you can reproduce this with third-party tool JQ which does JSON preprocessing.

$ cat cluster_health.json | jq -rc '{status}'
